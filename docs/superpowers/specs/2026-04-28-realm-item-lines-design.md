# 境界物品線總設計

## 背景

目前遊戲已經有基礎 `ItemCategory`、功法秘卷、裝備、丹藥、突破物、材料與 Workshop recipe。真正的缺口不是沒有物品系統，而是物品語彙還不夠支撐後續任務分支：低中境界材料種類偏少、任務物品沒有正式線、地區特產與貨幣代幣尚未被定義，導致任務容易退化成對話、擊殺與提交少量固定素材。

這份設計把物品拆成九條可獨立實作的物品線。每條線都要能回答：用途、來源、境界、職業/門派、消耗出口。

## 設計來源

修仙題材常見的物品系統包含丹藥、靈草、礦石、妖獸材料、法寶、符籙、陣盤、卷軸、傳承與貨幣。參考方向包含：

- 仙俠題材中的煉丹：靈草、礦物與藥引可煉成治療、修煉、破境或戰鬥輔助丹藥。
- 《鬼谷八荒》法寶與鍛造：高階法寶需要核心材料、天材地寶、世界 Boss 與強化素材承接。
- 《了不起的修仙模擬器》物品廣度：丹藥、材料、妖獸部件、法寶、符籙、陣法、卷軸、精魄、古物與食材都能成為修仙世界的物品語彙。
- 修仙 RPG 的境界限制：高階丹藥與材料應有境界需求，避免低階修士跨階段亂用高階資源。

## 全域原則

1. 物品不是單純收藏。每種正式物品都要至少有一個來源與一個消耗出口。
2. 境界分層要一致。每個境界至少要有丹材、器材、怪物材料、丹藥、裝備與任務物品的代表。
3. 職業差異要進入物品語彙。劍修偏劍鋼、劍魄、劍印；體修偏血骨、獸魂、臂甲；法修偏星蓮、魂露、法環。
4. 任務物品要和材料分開。任務信物、殘圖、玉簡不應被 Workshop recipe 誤消耗。
5. 圖鑑要能反查來源。物品線實作時必須保留 source hint 或可推導來源。
6. 第一輪先補資料骨架，不做法寶召喚、器靈養成或採集 UI 的大型 runtime。

## 九條物品線

### 1. 功法秘卷線

功法秘卷負責「學會功法」而不是圖鑑收藏。低階秘卷來自凡界藏經閣與門派任務；中階秘卷來自宗門藏經閣與精英怪；高階秘卷來自 Boss、秘境與傳承。每本秘卷必須能對應職業、境界、前置功法與至少一個來源。

### 2. 裝備線

裝備線負責戰鬥數值與職業 build。劍修使用劍、劍匣、劍印；體修使用拳套、臂甲、獸骨護符；法修使用法杖、法環、靈幡。通用防具維持頭、身、鞋、飾品、副手。高境界再承接法寶、器靈與帝兵。

### 3. 煉丹材料線

煉丹材料分為靈草、靈果、藥引、丹火媒介與毒材。每個境界要有可採集或掉落的丹材，並至少被一個丹方或任務消耗。

### 4. 煉器材料線

煉器材料分為礦石、精金、妖獸部件、器魂材料與天外材料。低階提供裝備重鑄；中階提供職業裝備；高階提供法寶、帝兵與路線材料 sink。

### 5. 丹藥線

丹藥分為修為丹、恢復丹、戰鬥丹、突破丹與特殊丹。修為丹支撐修煉，恢復丹支撐戰鬥，戰鬥丹提供短時間 build 變化，突破丹承接境界門檻，特殊丹提供壽元、悟性、福緣、抗劫或解毒。

### 6. 任務物品線

任務物品只服務任務，不進入一般煉丹煉器 recipe。它包含信物、令牌、薦書、妖丹樣本、殘破地圖、古修玉簡、NPC 私人物件與門派試煉印。

### 7. 地區特產線

地區特產讓不同地圖有自己的物品身份。仙緣鎮偏基礎資源，凌霄山偏劍鋼與寒鐵，萬獸谷偏血骨與獸魂，靈湖偏星魂蓮與月露，歸墟偏裂界晶與混沌殘核。

### 8. 貨幣代幣線

貨幣代幣承接不同商店與門派 loop。靈石是通用貨幣，宗門貢獻用於宗門商店，藏經閣書券用於秘卷兌換，試煉令用於門派試煉，傳承殘頁用於高階秘卷與法寶。

### 9. 符籙、陣盤與法寶線

符籙是一次性戰鬥/探索道具；陣盤支撐突破、洞府或戰鬥場域；法寶是高階裝備或召喚物；器靈是法寶成長線。第一輪只定義 taxonomy 與資料入口，不做複雜戰鬥 runtime。

## OpenSpec 拆分

本設計拆成九條 OpenSpec change：

1. `add-skill-manual-item-line`
2. `add-equipment-item-line`
3. `add-alchemy-material-line`
4. `add-smithing-material-line`
5. `add-pill-item-line`
6. `add-quest-item-line`
7. `add-region-specialty-item-line`
8. `add-currency-token-item-line`
9. `add-talisman-array-and-artifact-line`

第一批實作建議先做前五條，因為它們直接支撐現有功法、裝備、Workshop 與戰鬥補給。後四條先開規格，等資料骨架穩定後再串任務與地圖。

## Persistence

第一輪只新增 catalog item 與 authoring audit，不新增玩家存檔欄位。既有存檔只保存 `itemId/count/instance`，新增 item id 不需要 migration。若後續加入採集狀態、代幣餘額、法寶器靈養成或符籙快捷欄，才需要另開 persisted schema change。

## 圖鑑呈現

物品線完成後，萬界圖鑑的物品頁不應再只以「神兵法寶」概括所有物品。功法與裝備維持獨立入口：功法放在原本 `功法神通` tab，裝備放在獨立 `裝備法寶` tab。一般物品頁改為「萬物圖鑑」，並在境界分段上方提供物品線分類：丹藥、煉丹材料、煉器材料、任務物品、地區特產、貨幣代幣、符籙、陣盤、法寶器靈、突破物與其他。每張卡片保留原本 description、品質、來源追蹤與原始子類，額外顯示主分類，讓玩家能同時知道「這是什麼線」與「它原本的素材/裝備/消耗品子類」。

此呈現只讀取既有 item catalog 與 `itemLineMetadata`，不新增玩家存檔欄位，因此不需要 migration。

## 驗證策略

1. 每條物品線都有獨立 audit test。
2. 新增 item id 必須存在於 `ITEMS`。
3. 每個境界至少有對應代表物品。
4. 每個正式材料至少有 source hint 或 recipe / quest / drop 使用端。
5. Workshop recipe、商店與掉落表不得引用不存在的 item id。
6. `npx tsc --noEmit`、`npm test`、`npm run build` 作為完成 gate。
