# 三職業核心技能池草案

本文件的目的很直接：

把三職業的技能，從「每個大境界都塞一套」改成「有限、清楚、可擴充」的核心技能池。

這份文件原本是純草案，但目前已經有一版正式 registry 落到程式。  
重點是先把：

1. 每個職業到底需要哪些技能類型
2. 哪些技能適合商店賣
3. 哪些技能適合精英掉
4. 哪些技能應該是 Boss 核心書

全部先收斂清楚。

## 1. 整體原則

### 1.1 每個職業的技能總量

我建議先控制在這個範圍：

- 主動技能：`7`
- 被動技能：`5`

也就是單職業先抓：

- `12` 個核心技能

三職業總計：

- `36` 個核心技能

這個量比「每個大境界都發一套」合理很多，而且還足夠支撐長期成長。

### 1.2 技能來源分配

我建議每個職業的 12 個技能，先這樣分：

| 來源 | 數量 | 定位 |
| --- | ---: | --- |
| 商店 / 宗門基礎書 | 4 | 保底可玩核心 |
| 精英掉落書 | 4 | 中期功能補強 |
| Boss 核心書 | 2 | 強力主動 / 關鍵被動 |
| 傳承 / 任務 / 特殊兌換 | 2 | 高階變體或終局技能 |

這樣的好處：

1. 不會讓玩家卡在沒技能可用
2. 精英有農點
3. Boss 有真正的技能追求
4. 後期還保留擴充空間

### 1.3 境界的作用

境界不再負責「大量新增技能」，而改成：

- 解鎖技能書學習資格
- 解鎖更高階技能槽
- 讓舊技能進化或被替換

也就是：

- 練氣拿到職業骨架
- 築基 / 金丹補功能與範圍技
- 元嬰以上補終局技與變體

## 2. 劍修核心技能池

### 2.1 職業定位

- 攻速最快
- 單體輸出最強
- 破防、追擊、短 CD 爆發
- 對 Boss 最有壓力

### 2.2 主動技能

#### 1. 基礎劍訣

- 類型：單體
- 來源：商店
- 定位：最早期的穩定主攻技
- 功能：短 CD，單體高於普攻

#### 2. 疾風突斬

- 類型：突進 / 單體
- 來源：商店
- 定位：貼怪與切入
- 功能：短位移 + 傷害

#### 3. 破甲劍氣

- 類型：單體 / 破防
- 來源：精英掉落
- 定位：打精英與 Boss 的功能技
- 功能：降低敵方防禦或增傷窗口

#### 4. 劍輪橫掃

- 類型：扇形 / 小範圍
- 來源：精英掉落
- 定位：補群怪能力
- 功能：前方扇形斬擊

#### 5. 追影連斬

- 類型：單體多段
- 來源：精英掉落
- 定位：高機動連段
- 功能：連續數段傷害，適合中期主力輸出

#### 6. 萬劍歸宗

- 類型：範圍爆發
- 來源：Boss 核心書
- 定位：高辨識度的大招
- 功能：大範圍劍氣爆發

#### 7. 一劍斬命

- 類型：單體終結
- 來源：傳承 / 高階任務
- 定位：Boss 特化
- 功能：斬殺、低血增傷或高爆發長 CD

### 2.3 被動技能

#### 1. 劍心凝神

- 來源：商店
- 定位：基礎暴擊被動

#### 2. 御劍步

- 來源：商店
- 定位：攻速 / 移速 / 短 CD 補正

#### 3. 劍氣共鳴

- 來源：精英掉落
- 定位：疊層型增傷

#### 4. 破鋒真意

- 來源：Boss 核心書
- 定位：破防後增傷 / 暴傷放大

#### 5. 無我劍域

- 來源：傳承 / 高階兌換
- 定位：終局被動，讓劍修進入成熟流派

## 5. 技能取得節奏示意

### 練氣

- 商店：每職業 2 本基礎書
- 精英：每職業 1 本進階書
- Boss：每職業 1 本較強核心書

### 築基 / 金丹

- 商店 / 宗門：補 1 本新基礎書或基礎被動
- 精英：補 1~2 本功能 / 範圍書
- Boss：補 1 本核心強技

### 元嬰以上

- 不再大量增加技能數量
- 以：
  - 變體
  - 強化版
  - 終局核心書

為主

這樣才不會把技能池做爆。

## 6. 即時戰鬥下的對應方式

既然未來技能要進即時戰鬥，那這 3 個職業的技能池設計，本身就要能對應：

### 劍修

- 短射程
- 高攻速
- 快速切入
- 單體壓力高

### 體修

- 近身
- 慢一些
- 控場與承傷明顯
- 範圍技偏震盪型

### 法修

- 長射程
- 技能為主
- AOE / DOT / 控制明顯
- 普攻只是補充

所以我現在這份技能池草案，不只是平衡考量，也是直接為未來即時戰鬥鋪路。

---

## 7. 我對目前專案的實際建議

如果你要正式落地，我建議這樣做：

1. 先把現有技能表收斂到這份核心技能池
2. 把多餘或描述過於複雜的技能先暫時下線
3. 先做：
   - 商店書
   - 精英書
   - Boss 書
4. 再做即時戰鬥引擎需要的：
   - 射程
   - CD
   - 攻速
   - 範圍類型

## 8. 結論

這份草案的核心不是把技能做滿，而是把技能池收斂成：

- 數量可控
- 職業差異清楚
- 來源節奏合理
- 能直接接進即時戰鬥

如果你同意，我下一步就可以直接把現有 `data/skills/*.ts` 依照這份草案，開始做第一輪收斂與重組。
- retired alias 的正式 realm 視圖現在已統一由 alias-layer 聚合表組裝，skill index 不再分別攤開 active / passive alias。
- 所有 `getSkillsByRealm(...)` 的 realm dataset 目前都只保留 `core` 技能，retired skill 僅存在中央 alias / 相容查詢層。
