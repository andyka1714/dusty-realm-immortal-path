## 2. 尺寸與合理性原則

玩家角色固定作為 `1x2` 基準。怪物尺寸不套用固定 rank 表，而是由名稱、描述、body type、元素、rank 與地圖語境推導。

- 目前 player movement/combat 透明 sheet 的實測高度約 `80px`，腳底線約 `y=89`。
- 因此 `1x2` 可視為 `80px` 高，`1 tile` 可視為約 `40px`。
- 普通人形怪若為 `1x2`，movement 與 combat 的 actor 高度都必須接近 `80px`；combat 允許因武器、手臂或施法動作變寬，但不得變矮、變胖或變成大頭 Q 版。
- 同一怪物 movement/combat 平均高度差距不得超過 `4%`，腳底線差距不得超過 `1px`。
- 同一 movement sheet 內的 frame 高度差距應控制在 `2px` 內；combat 因攻擊姿態可放寬到 `4px` 內。
- 人形、劫匪、劍客、道童：通常維持 `1x2`，精英可提升到 `1x3`。
- 走獸、野狗、狼、熊、獸王：依體型多為 `2x1`、`2x2` 或 `1x3`。
- 蟹、龜、蜘蛛、蜈蚣、蠍：優先使用低伏橫向 footprint，例如 `2x1`、`3x2`、`4x2`。
- 蛇、蛟、龍：使用長條或盤繞 footprint，Boss 可提升到 `4x2`。
- 魂、靈、幽、鬼、劍意：以漂浮靈體高度表現，保留腳底 / shadow anchor。
- 劍尊、仙尊、天道化身、鴻蒙道祖：以人形法相或巨像呈現，不因 Boss 身分一律轉成伏地巨獸。

## 3. 圖樣生成策略

本 change 不允許用程式繪製 placeholder 充當完成圖檔。每個 enemy template 都必須透過 `$generate2dsprite` 流程獨立輸出：

- `public/assets/generated/characters/monsters/<enemy-id>-movement-v1/`
- `public/assets/generated/characters/monsters/<enemy-id>-combat-v1/`

每個資料夾包含：

- `raw-sheet.png`
- `raw-sheet-clean.png`
- `sheet-transparent.png`
- `frames/`
- `animation.gif`
- `prompt-used.txt`
- `pipeline-meta.json`
- `asset.json`

movement sheet 為 4x4，combat sheet 為 4x6，row order 固定為 `down / left / right / up`。

流程必須是：

1. 依怪物 profile 手寫 prompt。
2. 先完成 movement sheet，並把通過 QC 的 movement sheet 作為該 enemy 的 style reference。
3. 生成 combat sheet 時必須沿用 movement sheet 的 silhouette、palette、outline、scale、view angle 與 footline。
4. 使用 built-in `image_gen` 產生 raw sheet。
5. 使用 `$generate2dsprite` processor 只做 magenta cleanup、frame splitting、alignment、QC 與 transparent export。
6. frame QC 與 movement/combat pair style QC 都通過後，才可把 asset source 從 `pending_generate2dsprite` 改為 `generated`。

每次生成後都必須同時檢查：

- raw row sheet：每格的頭、尾、腳、武器、翅膀、角、特效都完整留在 cell 內。
- processed `sheet-transparent.png`：切格後沒有任何方向的身體部位被裁掉。
- individual frame PNG：單格檢查時仍保留完整輪廓，尤其是 `up/back` row 不得出現頭頂或尾端被截平。
- 數值 QC 只能證明大小與腳底線，不能取代視覺完整性檢查；若肉眼看到裁切，即使 `monster:qc` 通過也必須退回重生。

逐隻怪物完成度由 `docs/06_Balance_Audit/22_怪物圖樣生成Checklist.md` 追蹤。每完成一批圖樣後執行 `npm run monster:checklist` 重新產生 checklist，確認 movement / combat / production-ready 狀態沒有漏登。

同一 enemy 的 movement / combat 不得只「題材相近」。兩者必須像同一隻怪物在不同動作中延伸，否則即使 processor 沒有 edge-touch，也必須退回重新生成。

同一 enemy 的 movement / combat 也必須使用相同 normalized frame size。一般怪物與人形怪物預設為 96px frame；不得出現 movement 輸出為 96px、combat 輸出為 128px 的比例混用。人形怪物需參考 `public/assets/generated/characters/npcs` 與 player 的修長比例，避免生成大頭版或 combat 放大版角色。

人形與有腳怪物的 movement sheet 必須參考 player 的步態節奏：每個方向列第 1、3 格為站定 / 重心 frame，第 2、4 格分別呈現左右腳交替。若四格看起來只是重複站立、同一側腳、或連續相同圖示，該 movement sheet 必須退回重新生成。

非人形怪也要遵守相同節奏概念：

- 四足怪：第 1、3 格為中性站姿 / 重心，第 2、4 格為相反前後腳節奏。
- 伏地甲殼：第 1、3 格為收束，第 2、4 格為左右鉗、步足或身體重心相反 phase。
- 蛇 / 蛟 / 龍：第 1、3 格為中性波形，第 2、4 格為相反 S 曲線或身體波峰。
- 飛行：第 1、3 格為中性翅位，第 2、4 格為上拍 / 下拍或左右翼節奏相反。
- 靈體：第 1、3 格為收束輪廓，第 2、4 格為能量流、飄帶、法光相反 phase。

## 4. Rank 視覺語言

- Common：低干擾輪廓、標準 target cue、低動畫強度。
- Elite：元素邊光、較亮輪廓、elite target cue、中等動畫強度。
- Boss：domain aura、boss target cue、強血條 / telegraph 語言、高動畫強度。

Rank 只影響視覺強度與 UI cue，不改寫怪物物種語意。

## 5. Persistence

本輪不新增 LocalStorage persisted 欄位，也不改 hydrate shape。

怪物視覺資訊由 `Enemy.id` / `templateId` 回查 visual profile 與 asset registry；active monster instance 不保存 footprint、height 或 asset id。因此不需要 migration。
