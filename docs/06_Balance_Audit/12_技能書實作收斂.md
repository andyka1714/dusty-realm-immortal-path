# 技能書實作收斂

本文件記錄本輪已正式落地的技能書系統，避免分析文件與正式實作再次漂移。

## 1. 已完成實作

### 1.1 移除自動授予技能

以下流程已不再直接送技能：

- `setProfession()`
- `attemptBreakthrough()`

角色現在必須透過技能書學習技能。

### 1.2 技能書改為由技能資料自動生成

所有正式保留的 `core` 技能都會自動生成對應技能書：

- 物品 ID 規則：`manual_<skillId>`
- 道具類型：`ConsumableType.Manual`
- 使用效果：`learn_skill`
- 限制條件：
  - `requiredProfession`
  - `requiredRealm`

這代表：

- 不再需要手動維護一份獨立的空殼技能書表
- 正式核心技能新增後，技能書會同步出現
- `transition / legacy` 技能已不再進入正式技能書取得池
- 若舊存檔或舊道具仍引用退場技能，hydrate 與學習流程都會自動映射到 `replacementSkillId`
- 舊秘卷 item id 會在載入時升級成對應 formal core 秘卷，不再把 retired manual 直接帶進 store

### 1.3 第一版技能書來源規則

目前正式採用的規則是：

- 凡界藏經閣：販售三職業練氣期主動 / 被動入門秘卷
- 宗門藏經閣：販售本宗築基被動秘卷
- 宗門入門試煉任務：各自獎勵本職業練氣主動秘卷
- 精英怪：掉落同境界被動技能書
- Boss：掉落同境界主動技能書

### 1.3.1 第二輪來源層級收斂

目前技能書來源已正式拆成 acquisition layer：

- `基礎秘卷`
  - 凡界藏經閣
  - 宗門入門試煉
- `進階秘卷`
  - 宗門藏經閣
  - 同境界精英
- `核心秘卷`
  - 同境界 Boss
- `傳承秘卷`
  - 古修傳承殿

現在每一本正式技能書都會攜帶：

- acquisition tier
- 來源類型列表
- 職業 / 境界限制
- 前置技能列表

### 1.4 自動驗證

目前已新增自動檢查，驗證：

- 每一招正式 `core` 技能都有對應技能書
- 每一本正式技能書都至少有一個正式來源
- 每一招技能都已綁定正式技能池 metadata
- 每個職業都固定為 `12` 招核心技能（`7` 主動 + `5` 被動）
- 每一招 `transition / legacy` 技能都已退場並指向正式核心替代技能
- 每一本技能書都帶有 acquisition tier、來源類型與前置技能 metadata
- 宗門試煉獎勵會自動對齊正式技能池中的對應入門主動技能書

用意是避免後續裁技能、改掉落或改商店時，再次出現資料漂移。

### 1.5 技能書與正式技能池開始共用同一套約束

目前 `data/items/manuals.ts` 保留「技能書來源 registry」，`data/skills/pool.ts` 補上「正式技能池 registry」。

兩者分工如下：

- `manuals.ts`
  - 處理技能書 ID、分類、來源標籤與描述文案
- `pool.ts`
  - 處理核心 / 過渡 / 舊版技能狀態
  - 處理保底 / 功能 / 爆發 / 被動角色
  - 處理正式來源層級
  - 處理前置技能鏈

這樣後續如果要刪技能、合併技能、調整掉落，不需要再靠人工記憶同步文件。

### 1.6 正式技能池已開始實際裁剪

目前不是只有標記 `transition / legacy`，而是已正式做兩件事：

1. 商店 / 精英 / Boss / 傳承只再投放 `core` 技能
2. 每個退場技能都補上 `replacementSkillId`，明確指向被正式吸收的核心技能
3. 角色技能列表、hydrate migration、技能書學習與正式查詢 helper 都已自動正規化到 formal core 視角

### 1.7 可先購得、後參悟已正式成立

目前商店已不再因 `requiredRealm` 提前隱藏技能書。
玩家可以先買下未達境界的秘卷，但：

- 未滿足職業 / 境界 / 前置技能條件前不可參悟
- 背包與商店 tooltip 會明確顯示 acquisition tier 與限制
- 正式學習判定仍以 `consumeItem -> learn_skill` 流程為準

## 2. 為什麼先這樣收斂

目前技能資料量已很多，但即時戰鬥尚未正式上線。  
如果現在就擴成每個大境界三到五本全新技能，會讓數值與效果維護失控。

因此第一版實作先採取：

- 保留現有雙技能骨架（主動 / 被動）
- 完整做出技能書學習流程
- 把來源分成商店、精英、Boss 三層

這樣後續要裁技能、補技能、改技能時，系統底層不需要重做。

## 3. 後續可擴充方向

下一步若要繼續深化，優先順序應為：

1. `transition / legacy` 技能的正式退場與舊存檔遷移已完成
   現在正式技能 / 秘卷 / 獎勵視圖維持 `core only`，舊 skill id 與舊 manual item id 會在 hydrate 時升級到 formal core
2. 把高境界技能補成真正可實作的效果，而不是只靠描述文案
   目前已完成第六批高境界效果接線：`s_bi_passive`、`s_ma_passive`、`b_bi_active`、`b_bi_passive`、`m_bi_passive`、`m_ma_passive`、`s_im_passive`、`b_tr_passive`、`m_tr_active`、`m_tr_passive`、`b_im_passive`、`s_ie_passive`、`m_ie_active`、`m_ie_passive`、`b_ie_active`
   同時也已把正式核心高境界技能的描述文案收斂到目前 battle core 真正可承接的效果，不再保留「無視防禦 / 冷卻暫停 / 不可驅散 / 當前生命加成」這類未完整接線的說法
3. 後續若再新增技能書，優先保持和即時戰鬥的範圍、射程、CD 與狀態系統同步
