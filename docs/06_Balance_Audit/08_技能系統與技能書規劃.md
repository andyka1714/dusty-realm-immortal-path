# 技能系統與技能書規劃

本文件專門回答三件事：

1. 目前技能系統和你的設計目標哪裡衝突
2. 技能改成「技能書取得」後，應該怎麼設計
3. 每個境界、每個職業目前技能資料是否完整

## 1. 目前系統的真實狀況

### 1.1 現在有兩套互相衝突的技能取得邏輯

目前專案同時存在：

1. `consumeItem -> learn_skill`
2. `setProfession / attemptBreakthrough -> 自動授與技能`

對應檔案：

- `store/slices/characterSlice.ts`
- `data/items/manuals.ts`
- `types.ts`

這代表現在的狀況是：

- 系統底層其實已經支援「吃道具學技能」
- 但主流程卻又在選職業或突破時直接送技能

這和你現在要的設計是衝突的。

### 1.2 目前技能書資料其實還是空殼

`data/items/manuals.ts` 裡現有的「功法」：

- `吐納法`
- `鐵布衫`
- `輕身術`
- `靈力一擊`

它們現在只有描述，`effects` 幾乎是空的，還沒有正式對應到：

- `skillId`
- 對應境界
- 對應職業
- 商店取得 / 掉落取得

也就是說：

- 技能書概念存在
- 但還沒有真的成為主系統

### 1.3 目前三職業技能資料覆蓋

目前 `data/skills/*.ts` 已有：

- 練氣
- 築基
- 金丹
- 元嬰
- 化神
- 煉虛
- 合體
- 大乘
- 渡劫
- 仙人
- 仙帝

每個大境界，原則上三職業都有一組：

- 1 個主動
- 1 個被動

但有一個很重要的限制：

- 這些技能大部分是快速生成的概念資料
- 不少描述涉及的特殊效果，現在戰鬥引擎並沒有完整支援

例如：

- 劍氣疊層
- 冷卻重置
- 嘲諷
- 護盾
- 凍結
- 多段隨機目標
- 各種條件觸發被動

所以不能把現在這份技能表直接視為「可上線平衡版」。

## 2. 我對技能取得方式的建議

你的方向是對的：  
技能不應該在境界提升時自動送，而應該透過技能書取得。

我建議拆成三層。

### 2.1 基礎功法：商店販賣

定位：

- 保證每個職業都能取得一套基本可用技能
- 避免玩家因為掉不到書而斷流

適合放商店的技能：

- 練氣主動
- 練氣被動
- 築基的基礎被動
- 通用基礎功法

設計原則：

1. 便宜
2. 穩定取得
3. 只保證「能玩」，不保證「最強」

### 2.2 進階功法：精英怪 / Boss 掉落

定位：

- 用來形成真正的農裝與養成節奏
- 讓路線主題更明確

適合掉落的技能：

- 築基以上的職業主動
- 強力被動
- 特化流派技能
- Boss 專屬秘術

設計原則：

1. 普通怪低機率
2. 精英怪中機率
3. Boss 高機率或保底材料兌換

### 2.3 宗門專屬傳承：任務 / 聲望 / 兌換

定位：

- 不靠純掉率賭臉
- 提供一條可預期進度線

適合放這層的技能：

- 各宗門核心傳承
- 同職業的流派分支技能
- 高階被動或終局技能

## 3. 我建議的技能書結構

目前 `ConsumableEffect` 已支援：

- `learn_skill`

所以短期不需要重做整個物品系統。  
直接把技能書做成 `ConsumableType.Manual` 即可。

我建議每本技能書至少包含：

- `id`
- `name`
- `skillId`
- `requiredProfession`
- `requiredRealm`
- `sourceType`
  - `shop`
  - `elite_drop`
  - `boss_drop`
  - `quest`
  - `sect_exchange`

短期最實際的做法是：

1. 保留 `Manual` 當技能書分類
2. 在 `effects` 裡正式填上 `learn_skill`
3. 消耗時檢查職業與境界條件

## 4. 我對技能配置節奏的建議

### 4.1 每個大境界的基本配置

我建議每個職業每個大境界至少維持：

- 1 個基礎主動
- 1 個基礎被動
- 1 個較強的進階技能

其中「較強的進階技能」不應該走商店，而應該是該大境界的核心追求。

但不要全部都靠掉落。

比較合理的是：

| 境界 | 基礎技能 | 進階技能 | 核心強技 |
| --- | --- | --- | --- |
| 練氣 | 商店 | 精英怪低機率 / 宗門商店 | Boss 掉落 |
| 築基 | 宗門商店 | 精英怪掉落 | Boss 掉落 |
| 金丹 | 宗門商店 / 任務 | 精英怪掉落 | Boss 掉落 |
| 元嬰 | 任務 / 宗門兌換 | 精英怪掉落 | Boss 掉落 |
| 化神以上 | 傳承 / 兌換 | 精英怪掉落 | Boss / 特殊首領掉落 |

也就是說，每個大境界都應該有：

1. 一套保底可取得的基本技能
2. 一套精英怪可以掉的進階技能
3. 一套 Boss 才會掉的核心強技

這樣技能養成才會真的和怪物挑戰、裝備節奏綁在一起。

### 4.2 我建議的技能書掉落分層

如果套到你現在的遊戲節奏，我建議這樣設計：

#### 普通怪

- 不掉核心技能書
- 最多只掉：
  - 通用基礎功法殘頁
  - 極低機率的路線基礎技能書

#### 精英怪

- 該境界一定要有可掉的職業技能書
- 但應偏向：
  - 進階主動
  - 進階被動
  - 流派補強技能

這一層的作用是：

- 讓玩家在打 Boss 前就有一條可農的技能成長線
- 不會完全只看裝備

#### Boss

- 每個大境界都至少要有 1 本較強的核心技能書
- 這本書應該是：
  - 比同境界商店技能更強
  - 比精英技能更有辨識度
  - 能明顯改變戰鬥節奏

這樣 Boss 的價值才不只是掉裝備。

### 4.3 每個大境界的技能配置模板

我建議每個職業每個大境界，最終都收斂成下面這種模板：

| 類型 | 數量 | 來源 |
| --- | ---: | --- |
| 基礎主動 | 1 | 商店 / 宗門 |
| 基礎被動 | 1 | 商店 / 宗門 |
| 進階主動 | 1 | 精英怪 |
| 進階被動 | 1 | 精英怪 |
| 核心強技 | 1 | Boss |

這樣一個大境界、單一職業，完整會有 5 本技能書。

優點：

1. 有保底
2. 有農點
3. 有 Boss 追求
4. 不會只剩裝備在成長

### 4.4 三職業的技能設計方向

#### 劍修

- 核心：單體爆發、破防、追擊、加速循環
- 商店技能：穩定單體斬擊、基礎暴擊被動
- 掉落技能：劍氣疊層、斬殺、冷卻重置、穿甲

#### 體修

- 核心：站場、反擊、減傷、控制
- 商店技能：穩定重擊、基礎護體被動
- 掉落技能：反震、嘲諷、護盾、血量越低越強

#### 法修

- 核心：技能輸出、元素狀態、資源循環
- 商店技能：單體法術、基礎回靈被動
- 掉落技能：AOE、DOT、凍結、連發、減 CD、回藍

## 5. 對目前技能資料的審核結論

### 5.1 可以保留的部分

目前技能資料中，這些可以保留：

- 各境界有主動 / 被動的基本骨架
- 三職業定位方向大致正確
- 名稱與題材感大多沒問題

### 5.2 不能直接照用的部分

以下不能直接視為完成版：

1. 很多技能描述比實作能力更複雜
2. 被動技能雖已開始走逐招明確收益與專屬效果，但仍有少數高境界被動尚未完全專屬化
3. 很多 `statusEffect` 仍未完整支援
4. 技能取得方式已正式接進商店 / 掉落 / 傳承，但文件仍需持續對齊實作細節

所以現在正確的方向不是「再多生技能」，而是：

1. 先把技能取得邏輯改正
2. 再把現有技能分成：
   - 可直接上線
   - 需要簡化後上線
   - 保留概念、未來再做

## 6. 目前技能資料離這個目標還差什麼

目前雖然每個大境界三職業原則上都有 1 主動 + 1 被動的骨架，但離上面的模板還差很遠：

1. 沒有正式區分「基礎 / 進階 / Boss 核心強技」
2. 各大境界的技能書來源雖已綁到商店 / 精英 / Boss / 傳承，但還要把剩餘 retired / duplicate 技能再進一步裁剪
3. 高境界欄位與效果仍有尾端需要補完
4. 不少技能描述仍偏複雜，需要持續收斂到當前引擎可完整承接的形式

所以後面正確做法不是「直接全開」，而是要先把每個境界的技能書層級做出來。

## 7. 我建議的短期實作順序

### 第一階段

1. 拿掉職業 / 突破自動送技能
2. 把練氣三職業技能改成技能書
3. 補齊宗門技能商店
4. 為每個大境界建立 `elite manuals` 與 `boss manuals` 的掉落表

### 第二階段

1. 築基、金丹技能改成怪物 / Boss 掉落
2. 把技能書和掉落池對應到三路主題
3. 補技能學習條件檢查
4. 幫每個大境界各職業補一個 Boss 核心強技

### 第三階段

1. 逐招重審技能描述與實作可行性
2. 把剩餘少數仍偏通用模板的被動升級成專屬效果
3. 再做真正的流派分化

---

## 8. 結論

如果要符合你要的修仙體驗，技能系統最重要的不是「現在能不能打出技能」，而是：

1. 技能取得要合理
2. 技能來源要有成長節奏
3. 技能本身要和職業 / 地圖 / 掉落主題一致

所以我認為下一步最應該做的，不是再送技能，而是正式把技能改成技能書系統。
