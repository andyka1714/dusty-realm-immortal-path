# 職業與技能審計

本文件聚焦三件事：

1. 三職業現在到底有沒有真的成立
2. 技能目前接到了哪裡
3. 還剩哪些缺口沒有補完

---

## 1. 目前職業系統的真實現況

### 1.1 已經成立的部分

這次修正後，三職業已經不只是資料表：

1. `ProfessionType` 會影響小境界成長
2. 技能池已正式收斂到 formal core，並透過技能書學習
3. 戰鬥會依職業切換輸出模式
4. 不同職業的 Boss 門檻已能靠測試驗證

對應檔案：

- `store/slices/characterSlice.ts`
- `utils/battleSystem.ts`
- `data/skills/index.ts`

### 1.2 原本最嚴重的缺口

原本真正的問題其實有三個：

1. 小境界成長一直套用 `"None"`，職業成長沒生效
2. 主動技能沒有進戰鬥
3. 練氣期職業技能資料根本不存在

這三個現在都已補上第一版。

---

## 2. 技能系統目前已接上的部分

### 2.1 主動技能

目前已支援：

- 讀取已學技能
- 挑選當前境界可用的最高主動技能
- 冷卻
- MP 消耗
- 技能戰鬥日誌
- 技能倍率
- `stun / burn / poison / bleed / shield / armorBreak / reflect` 等正式狀態
- world strike / timeline combat 共用的 status resolver 與 cooldown / timing metadata

這代表：

- 劍修不再只是純平砍
- 體修不再只靠防禦堆傷
- 法修不再只是假裝有 `magic`

### 2.2 被動技能

目前被動技能已不再只靠一層「職業通用增益」：

- formal core 被動的基礎收益已改成逐招明確對照表
- 多招高境界被動已補上專屬事件、保命、免疫、回復或冷卻互動
- absorbed retired passive 透過 formal id 承接時，也會吃到同一套明確對照

目前仍未完全做到的是：

- 每一招被動都具備完整的專屬觸發與專屬戰鬥事件
- 少數高境界被動仍偏框架型，雖已補齊部分開場 / world strike 可見性，但還沒全部脫離共用模板

也就是說：

- 現在已經是「可用、可平衡、且開始逐招對齊描述」
- 但還不是「所有被動都完全結案的最終版技能設計」

## 3. 練氣期補齊內容

這次新增：

### 劍修

- `疾風三疊`
- `劍脈初成`

定位：

- 讓北路在練氣期就能開始走單體爆發與破防節奏

### 體修

- `裂地重拳`
- `銅皮鐵骨`

定位：

- 讓西路偏穩定站場，但避免太早壓垮 Boss 難度

### 法修

- `玄水飛芒`
- `靈潮循環`

定位：

- 讓東路在分流期就有明確法傷輸出入口

---

## 4. 三職業目前的建議定位

### 劍修

- 單體爆發最高
- 打 Boss 最有存在感
- 需要靠暴擊與技能窗口拉開差距

### 體修

- 容錯最高
- 普通怪與精英最穩
- Boss 應該是「能磨但不該太早穩吃」

### 法修

- 最依賴技能
- 對普通怪與多段輸出表現最好
- 裝備起來後越階能力應最明顯

## 5. 已驗證的門檻

這次已用 `utils/battleSystem.test.ts` 驗證：

1. 練氣三職業全套下品可以清同路普通怪
2. 練氣三職業全套下品不能穩吃同路 Boss
3. 練氣三職業全套中品 + 4 件上品可以挑戰同路 Boss

這代表目前至少已經有：

- 可驗證的開荒帶
- 可驗證的 Boss 門檻
- 可驗證的三職業差異

---

## 6. 尚未補完的技能缺口

### 6.1 高境界技能欄位與事件已收口

目前所有被動技能都已退出 generic `passiveEffectTags` fallback，`passiveEffectTags` 欄位本身也已自資料層移除；這輪也把 `獸神附體` 這類高境界變身型 active 正式接回 shared status / logger / world strike / timeline / replay 路徑，所以先前文件裡提到的「少數高境界欄位與戰鬥事件仍待補完」已可視為收口。

後續如果要更進一步：

1. 針對更少數的技能描述做文字收斂，而不是再補 battle 缺欄位
2. 視需要再為個別高境界技能增加更細的環境互動或專屬演出
3. 繼續把 `world strike / timeline combat / replay` 的 status 表現維持在同一套語意之下

### 6.2 狀態效果仍偏少

目前實際支援已不只 `stun`，也已正式接入：

- `freeze`
- `paralyze`
- `banish`
- `burn`
- `poison`
- `bleed`
- `shield`
- `armorBreak`
- `reflect`
- `vulnerable`
- `taunt`
- `獸神附體`

後續應依序補：
- 更多怪物 / 玩家專屬控制與環境互動狀態

---

## 7. 這份文件的結論

現在的職業與技能系統，已經從「只有資料」進展到「可以進戰鬥、可以驗證、可以平衡」。

但還沒到最終狀態。

下一步最合理的方向不是再亂加技能數量，而是：

1. 先補更多怪物 / 玩家專屬控制與環境互動狀態
2. 再持續收斂技能描述與技能書層級
3. 最後才去做更複雜的流派與套路
