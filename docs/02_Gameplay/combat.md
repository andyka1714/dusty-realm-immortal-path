# 戰鬥流程 (Combat Mechanics)

## 1. 目前戰鬥循環 (地圖內直接出手 + 世界內回擊 + 相容時間軸驗證)
目前正式實作已進到第一版地圖內即時戰鬥：玩家在地圖上鎖定目標後，可直接普攻或施放主動術式；怪物也會在場景內依射程直接回擊。舊的 `runAutoBattle()` 與戰報流程仍保留作為數值驗證與相容尾端，不再是主要遊玩路徑。

1. **遇敵**: 玩家進入怪物所在的 Grid。
   - 劍修 / 體修：接戰距離 `1`
   - 法修：接戰距離 `3`
   - 部分遠程怪物已支援較長的 `attackRange`
   - 玩家鎖定目標後，只要進入可攻擊距離即可直接在場景上出手，不必再貼到同格
   - 怪物也會依自己的攻擊距離、索敵距離與 AI 類型在場景內回擊
2. **世界內交手**:
   - `F` 直接普攻
   - `Q` 直接施放主修主動術式
   - `R` 可開關掛機，進入距離後自動出手
   - 玩家與怪物的血量、護盾、狀態與技能 CD 會在場景 HUD 直接更新
3. **時間軸驗證內核**:
   - 根據角色職業、裝備、靈根、已學技能建立戰鬥用屬性。
   - 雙方在時間軸上依攻速與技能秒 CD 進行攻擊。
   - 主動技能會消耗真元並觸發特殊效果。
   - 被動技能會提供戰鬥內的常駐修正。
   - 舊戰報回放仍依事件 `timeMs` 驅動，但已退成相容 / 驗證用途。
4. **勝負判定**:
   - 怪物 HP <= 0: 勝利，獲得掉落與經驗。
   - 玩家 HP <= 0: 在地圖上直接判定重傷並傳送回安全區。

## 2. 技能接線狀態

- 技能已正式參與戰鬥結算
- 技能取得方式為技能書學習，不再隨選職業或突破自動授予
- 技能書學習現在會正式檢查前置技能鏈，不符合者無法參悟
- 背包中的技能書現在已可顯示：
  - 技能階級
  - 主動 / 被動分類
  - 主要取得來源（商店 / 精英 / Boss / 傳承）
- 技能書來源已整理成正式 registry，後續背包、圖鑑與文件都應共用同一份來源資料，不再分散在描述文字中
- 技能池也已有正式 registry，包含：
  - `poolStatus`
  - `formalRole`
  - `formalSourceTier`
  - `prerequisiteSkillIds`
- 正式可取得技能目前只保留 `core`：
  - `transition / legacy` 技能仍保留資料相容
  - 但已退出正式技能書 / 商店 / 掉落 / 傳承池
  - 並補上 `replacementSkillId` 指向被正式吸收的核心技能
- 法修會正式使用 `magic / mp / skill` 進行輸出，不再只靠物攻
- 主動技能資料已補上即時戰鬥所需欄位：
  - `castRange`
  - `cooldownSeconds`
  - `castTimeMs`
  - `projectileSpeed`
  - `areaShape / areaRadius`
- 練氣、築基的主動技能已手動補齊；更高境界主動技能會在匯出技能表時自動補齊即時戰鬥欄位，避免資料缺洞
- 玩家在地圖上的接戰距離，已開始吃「職業基礎距離 + 已學主動技能射程」
- `runAutoBattle()` 已從純回合式結算改為時間軸模擬，用來承接後續真正的同場即時戰鬥
- `castTimeMs` 與 `projectileSpeed` 已正式進入戰鬥節奏計算
- `burn`、`poison`、`bleed`、`shield`、`armorBreak` 已正式接進時間軸戰鬥內核
- `areaShape / areaRadius / maxTargets` 已正式進入目前的單體戰鬥判定，用來影響範圍技能在單體對決時的承傷效率
- 地圖內世界戰鬥已開始使用同一批欄位做第一版多目標命中：`circle / line / cone` 技能會依玩家與主目標的相對位置，波及附近其他怪物，而不再只是播放地面提示
- 即時戰鬥表現層的投射物、施法圈、AOE 與命中特效參數，已抽成可測的 presentation helper，後續調整不必只靠人工目測
- 世界戰鬥、時間軸戰鬥與右下角技能 CD HUD，現在已開始共用同一套技能冷卻解析，不再各自直接讀 `cooldownSeconds`
- 被動技能開關判定也已抽成共用 skill-id 對照表，world strike 與 timeline combat 對齊時不必再重複鋪一大串 `hasPassiveSkillId(...)`
- 戰鬥開場的被動狀態與對應戰鬥訊息，已開始走共用初始化 helper，不再由 `runAutoBattle()` 散寫 `荊棘皮層` / `元素護盾` 等開場事件
- 屬性克制、弱點洞察、元素抗性、開場被動與高境界開場壓制提示，也已開始收斂到共用 opener helper，降低 `runAutoBattle()` 前置事件散落風險
- 主動術式後的被動觸發訊息，已開始抽成共用 passive-proc helper，逐步收斂 `法力源泉` / `言出法隨` / `法則之劍` / `劍道獨尊` / `靈潮循環` / `蠻荒血脈` 等事件的散寫
- 來襲傷害的防禦型被動事件，也已開始抽成共用 defensive-passive helper，集中處理 `蠻荒血脈` / `銅皮鐵骨` / `金剛法相` / `肉身成聖` / `元素護盾`
- 第一批與第二批法術型怪物、遠程怪與 Boss 已補上 `specialAttack` 資料，戰鬥中不再只剩普通攻擊模板
- `createEnemy()` 已加入預設怪物特招規則：
  - 沒有手工特招的精英 / Boss 也會依 `元素 / 階級 / 境界` 自動得到一套基礎殺招
  - 已手工配置的怪物仍保留個別特化設定，不會被預設規則覆蓋
- 怪物現在已正式支援：
  - `resistances`
  - `weaknesses`
  - `affixes`
- 玩家使用對應元素攻擊時，會吃到怪物弱點 / 抗性乘區：
  - 弱點：直接傷害提高
  - 抗性：直接傷害降低
- 第一版怪物詞綴已正式進戰鬥：
  - `強襲`：提高怪物輸出
  - `堅甲`：提高怪物有效防禦
  - `霸體`：降低承傷，並有機率免疫控制型狀態
  - `統御`：同時提高攻防
  - `噬生`：命中後吸收部分傷害回復氣血
  - `迅影`：提高怪物攻擊節奏
- 法術型 / 遠程怪在地圖接戰前，已開始有別於近戰怪的追擊節奏；進入有效射程後會優先維持距離與側移，而不是一律貼臉追擊
- 築基三路 Boss 已補上 deterministic 門檻驗證：
  - 低裝同境界仍應卡關
  - 中裝搭少量高品裝應可挑戰通過
  - 劍修 / 體修 / 法修三條分流線已分別校正到這個區間
- 主場景底部已補上第一版戰鬥快捷列，可直接執行接戰、主動術式、掛機與地圖操作
- 舊的中央戰報窗已退場，戰鬥結果改為右下角 HUD 短暫顯示後自動收起
- 所有被動技能現在都具備 `passiveEffectTags`，後續逐招專屬效果與 HUD / 文件可共用同一套標記資料
- 已完成專屬接線的被動，開始移除手寫 generic `passiveEffectTags`，避免資料層繼續誤導成仍主要依賴通用加成

## 3. 後續規劃

- 後續會將目前的自動結算保留作為數值驗證工具
- 地圖內戰鬥將逐步改造成即時戰鬥
- 目前已完成第一階段的「距離接戰」基礎，下一步才是把實際攻擊動作、技能 CD、血條變化做成同場即時播放
- 即時版本預計引入：
  - 完全移除 battle modal 後的同場傷害數字
  - 角色 / 怪物血條
  - 投射物飛行與命中特效
  - 範圍提示圈
  - 更完整的狀態圖示
